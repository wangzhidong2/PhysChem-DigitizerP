## 🔌 接线指南

### ESP32-S3 接线方式

```
ESP32-S3 开发板       pH 放大模块
──────────────        ───────────
   5V             →      V
   GND            →      G (任一GND引脚)
   GPIO1          →      PO (模拟输出)
   GND            →      G (另一个GND引脚，增强稳定性)
```

**固件代码**: [ph esp32.ino](ph esp32.ino)

**引脚定义** (来自固件):
```cpp
#define ADC_PIN 1        // 使用 GPIO1 (ADC1_CH0)，ESP32-S3 推荐引脚
#define SAMPLE_INTERVAL 100  // 采样间隔 100ms (10Hz)
```

### ⚠️ 接线注意事项

1. **供电电压**：pH 放大模块需要 5V 供电，请勿接 3.3V
2. **接地**：建议连接两个 GND 引脚以增强稳定性和抗干扰能力
3. **PO 引脚**：模拟信号输出，连接到 ESP32-S3 的 ADC 引脚（GPIO1）
4. **电极保护**：pH 电极玻璃泡易碎，使用时请小心，不用时浸泡在保存液中

## 🧪 校准方法

### 重要说明

**与传统模块不同**：本项目采用 **Python 程序内校准**，校准参数保存在统一的 `sensor_config.json` 中，而非模块本身。

### 多模式校准原理

校准对话框支持 **单点 / 两点 / 三点** 三种校准模式，UI 会根据所选模式动态调整：

| 校准模式 | 拟合方法 | 数学模型 | 适用场景 | 所需缓冲液 |
|----------|----------|----------|----------|------------|
| **单点校准** | Nernst 理论斜率 | `pH = pH_ref + slope × (V - V_ref)` | 快速粗略校准，已知理论斜率时 | 1 个（任意 pH） |
| **两点校准** | 线性拟合 | `pH = k·ADC + b` | 一般测量场景，覆盖常用 pH 范围 | 2 个（如 pH 4.00 + pH 9.18） |
| **三点校准** | 二次多项式拟合 | `pH = a·ADC² + b·ADC + c` | 高精度测量，非线性补偿 | 3 个（如 pH 4.00 + 6.86 + 9.18） |

- **单点校准**：使用 Nernst 理论斜率（-59.16 mV/pH @25°C），只需一个参考点即可快速建立校准关系
- **两点校准**：通过线性拟合确定斜率和截距，适用于大多数常规测量
- **三点校准**：二次多项式拟合可补偿传感器非线性特性，精度最高
- 支持 **ADC 原始值** 或 **电压值** 输入，适配带信号调理的传感器模块

### 标准缓冲液推荐

| 缓冲液类型 | pH 值 | 推荐成分 |
|------------|-------|----------|
| 酸性 | 4.00 | 邻苯二甲酸氢钾缓冲液 |
| 中性 | 6.86 | 混合磷酸盐缓冲液 |
| 碱性 | 9.18 | 硼砂缓冲液 |

### 校准步骤

1. **选择校准模式**：
   - 点击"✏️ 编辑校准参数"按钮
   - 在校准对话框中选择校准模式（单点 / 两点 / 三点）
   - UI 将根据所选模式自动显示对应数量的输入框

2. **准备标准缓冲液**：根据所选模式准备相应数量的标准缓冲液

3. **打开上位机程序**：运行 `main.py`，选择 pH 传感器模块

4. **连接串口**：选择正确的 COM 口并连接

5. **开始采集数据**：点击"开始采集"按钮

6. **测量各缓冲液的 ADC 值**：
   - 将 pH 电极清洗后放入第一个缓冲液，等待读数稳定，记录 ADC 值
   - 若为多点校准，依次放入其余缓冲液并记录各 ADC 值

7. **编辑校准参数**：
   - 点击"✏️ 编辑校准参数"按钮
   - 选择校准模式后，输入各缓冲液的 pH 值和对应的 ADC（或电压）值
   - 点击确定保存

8. **保存配置**：配置会自动保存到 `sensor_config.json`，下次启动自动加载

### 校准示例

**三点校准示例**：
```
pH 4.00 → ADC 2555
pH 6.86 → ADC 2281
pH 9.18 → ADC 2030
```

**两点校准示例**（酸性 + 碱性）：
```
pH 4.00 → ADC 2555
pH 9.18 → ADC 2030
```

**单点校准示例**（中性参考点 + Nernst 斜率）：
```
pH 7.00 → ADC 2300  （仅需要一个参考点，其余由理论斜率计算）
```

## 💻 固件说明

### 数据输出格式

**串口输出**：`时间戳(ms),ADC原始值`

**示例数据**：
```
1234,2555
1334,2553
1434,2556
```

**ADC 配置**：
- 分辨率：12位（0-4095）
- 衰减：11dB（0-3.3V量程）
- 采样率：10Hz

### 固件核心代码

```cpp
#define ADC_PIN 1        // GPIO1 (ADC1_CH0)
#define SAMPLE_INTERVAL 100  // 100ms 采样间隔

void setup() {
  Serial.begin(115200);
  analogReadResolution(12);           // 12位精度
  analogSetAttenuation(ADC_ATTEN_DB_11); // 11dB衰减
  pinMode(ADC_PIN, INPUT);
  Serial.println("START");
}

void loop() {
  if (millis() - lastSampleTime >= SAMPLE_INTERVAL) {
    int adcValue = analogRead(ADC_PIN);
    Serial.print(millis());
    Serial.print(",");
    Serial.println(adcValue);
  }
}
```

## 🎯 使用说明

### 上位机功能

1. **实时数据显示**：显示当前 pH 值和 ADC 原始值
2. **数据曲线**：实时绘制 pH 值变化曲线
3. **统计信息**：显示平均值、最大值、最小值、标准差
4. **数据保存**：保存为 CSV 格式（包含时间、ADC、pH值）
5. **可配置采样率**：支持调整采样频率（默认 10Hz）
6. **校准参数编辑**：支持在程序内编辑校准参数

### 电极保养

1. **首次使用**：将电极在 3mol/L KCl 溶液中浸泡 2 小时活化
2. **日常保存**：短期保存浸泡在保存液或 3mol/L KCl 溶液中
3. **长期保存**：吸干电极内液，套上保护帽，干燥保存
4. **清洗**：测量前后用去离子水清洗电极，并用滤纸轻轻吸干

## ⚠️ 常见问题

### Q1: pH 值跳动很大
**原因**：电极接触不良、电源干扰、电极未稳定
**解决**：
- 检查接线是否牢固
- 使用稳定的电源
- 确保电极完全浸没在溶液中
- 等待读数稳定后再记录

### Q2: 测量值偏差大
**原因**：电极老化、校准不准确、温度影响
**解决**：
- 重新进行校准（建议使用三点校准以获得更高精度）
- 检查电极是否需要更换
- 考虑温度补偿（本模块暂不支持自动温度补偿）

### Q3: ADC 值异常（0 或 4095）
**原因**：接线错误、模块损坏、电极开路
**解决**：
- 检查 VCC、GND、PO 连接
- 用万用表测量 PO 引脚电压（应在 0-3.3V 之间）
- 检查电极是否连接良好

### Q4: 找不到串口
**原因**：驱动未安装、USB 线问题、端口被占用
**解决**：
- 安装 CH340/CP2102 驱动
- 更换 USB 线
- 关闭其他可能占用串口的程序
